# 병원 예약 시스템 (Hospital Reservation System)

> DDD, 클린코드, 디자인패턴을 학습하기 위한 병원 예약 시스템

## 📋 프로젝트 개요

복잡한 비즈니스 규칙을 가진 도메인 모델링 연습과 클린 아키텍처 적용을 위한 실무 수준의 백엔드 시스템입니다.

### 주요 기능
- 환자의 병원 예약/취소/변경
- 의사별 진료 스케줄 관리
- 예약 정책 및 제약사항 처리
- 대기 목록 및 알림 시스템
- 리뷰 및 평가 시스템
- 관리자 대시보드 및 통계

## 🛠 기술 스택

- **언어**: Java 17+
- **프레임워크**: Spring Boot 3.x
- **데이터베이스**: PostgreSQL
- **캐시**: Redis
- **메시지큐**: RabbitMQ
- **테스트**: JUnit 5, Mockito, Testcontainers
- **빌드**: Gradle

## 🏗 아키텍처

### 핵심 도메인
- **Patient**: 환자 정보 및 노쇼 기록 관리
- **Doctor**: 의사 정보 및 진료 스케줄 관리
- **Appointment**: 예약 생성/변경/취소 및 상태 관리
- **Waitlist**: 대기 목록 관리
- **Review**: 리뷰 및 평가 시스템

### 예약 상태 전이
```
PENDING → CONFIRMED → COMPLETED
    ↓         ↓
CANCELLED   NO_SHOW
```

## 📝 주요 비즈니스 규칙

### 예약 생성
- 예약은 1일 전부터 30일 후까지 가능
- 환자는 같은 날짜에 최대 2개 예약 가능
- 초진 30분, 재진 15분 시간 할당
- 의사 휴가/점심시간 예약 불가

### 예약 변경/취소
- 예약 24시간 전까지만 변경 가능 (최대 2회)
- 24시간 이내 취소는 노쇼 처리
- 노쇼 3회 누적 시 30일간 예약 제한

### 대기 목록
- 예약 마감 시간대에 대기 등록 가능
- 선착순(FIFO) 방식
- 취소 발생 시 대기 1순위에게 알림
- 30분 내 예약하지 않으면 다음 순위로 이동

## 🚀 시작하기

### 사전 요구사항
```bash
- Java 17 이상
- Docker & Docker Compose
- Gradle 7.x 이상
```

### 실행 방법
```bash
# 1. 의존성 설치
./gradlew build

# 2. Docker 컨테이너 실행
docker-compose up -d

# 3. 애플리케이션 실행
./gradlew bootRun
```

### API 문서
애플리케이션 실행 후 다음 URL에서 API 문서를 확인할 수 있습니다:
- Swagger UI: `http://localhost:8080/swagger-ui.html`
